# رفع البرنامج على Neon وRender

## أولًا: نسخ رابط Neon
1. افتح مشروعك في Neon.
2. افتح صفحة **Connect**.
3. اختر الاتصال من نوع **Pooled connection** إن ظهر.
4. انسخ قيمة رابط الاتصال الذي يبدأ غالبًا بـ:
   `postgresql://`
5. لا ترسل هذا الرابط لأي شخص؛ لأنه يحتوي على كلمة المرور.

## ثانيًا: رفع الملفات إلى GitHub
Render يحتاج مستودع Git غالبًا للنشر التلقائي.

1. أنشئ حساب GitHub إن لم يكن لديك.
2. أنشئ مستودعًا جديدًا باسم:
   `professional-accounting-erp`
3. اجعله **Private**.
4. ارفع محتويات هذا المجلد إلى المستودع، وليس ملف ZIP نفسه.

## ثالثًا: إنشاء Web Service في Render
1. من لوحة Render اضغط:
   **New > Web Service**
2. اربط حساب GitHub.
3. اختر مستودع:
   `professional-accounting-erp`
4. استخدم:
   - Runtime: Python
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn app:app`
   - Plan: Free

## رابعًا: إضافة متغير قاعدة البيانات
داخل Render:
1. افتح **Environment**.
2. أضف متغيرًا باسم:
   `DATABASE_URL`
3. ألصق رابط الاتصال من Neon في القيمة.
4. أضف متغيرًا باسم:
   `SECRET_KEY`
5. ضع قيمة طويلة وعشوائية، أو دع Render ينشئها عند استخدام Blueprint.

## خامسًا: التشغيل
بعد اكتمال النشر افتح رابط Render.

بيانات الدخول الأولية:
- Username: `admin`
- Password: `admin123`

يجب تغيير بيانات الدخول قبل إدخال بيانات حقيقية.

## تنبيه
هذه نسخة تجربة. لا تدخل بيانات مالية حقيقية أو حساسة قبل إضافة:
- تشفير كلمات المرور
- صلاحيات تفصيلية
- سجل تدقيق
- نسخ احتياطي
- حماية CSRF
- الربط النظامي مع ZATCA
